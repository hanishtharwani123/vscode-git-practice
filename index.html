1. Repository Setup

Create a new repository on GitHub without initializing README, then:

üåø Working with Branches

Creating and Switching Branches

In VS Code:

  1. Click the branch icon in the bottom left
  2. Select "+ Create new branch"
  3. Enter branch name (e.g., "feature/login")
git checkout -b feature/login
git push -u origin feature/login

üíæ Making Changes

Staging and Committing

In VS Code:

  1. Make your changes in the editor
  2. Open the Source Control panel (Ctrl+Shift+G)
  3. Click + to stage changes
  4. Enter commit message
  5. Click ‚úì to commit
Pro Tip: Write meaningful commit messages that explain WHY the change was made, not just WHAT was changed.

üîÑ Merging and Pull Requests

Creating a Pull Request

  1. Push your changes to your feature branch
  2. Go to GitHub repository
  3. Click "Compare & pull request"
  4. Add description and reviewers
  5. Click "Create pull request"
